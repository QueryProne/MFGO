Guardrails (Do Not Violate Without User Approval)
- Preserve existing architecture and module boundaries.
- Do not make drastic schema changes or destructive migrations without explicit approval.
- Prefer additive, backwards-compatible updates.
- Keep custom forms + fields + page assignment flows stable.

Operational Notes
- Primary local stack:
  - API: http://localhost:3000
  - UI:  http://localhost:5173
  - Postgres: docker container `mfgos-pg` on localhost:55432
- If UI shows many `/api/custom/*` 500 errors, verify API process is actually running on port 3000 first.
- Vite proxy (`:5173/api/*`) returns 500 when backend target is down/unreachable.

Current Known Behaviors
- Custom data types are backend-seeded and auto-bootstrap from `/api/custom/data-types`.
- Custom field creation requires selecting a data type and creates fields as required by default.
- Forms can be attached to pages via:
  - POST `/api/custom/entities/page/{pageId}/forms`
  - Example pageId: `administration`
