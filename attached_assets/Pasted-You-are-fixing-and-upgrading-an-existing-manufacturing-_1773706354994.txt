You are fixing and upgrading an existing manufacturing ERP codebase.

This is not a greenfield build. You must preserve the current project structure where reasonable, but refactor the schema, API, services, and UI to support proper industrial manufacturing workflows.

==================================================
PRIMARY GOAL
==================================================

Refactor the current ERP so that:

1. MRP / Planning and Purchasing operate as one integrated module and one operational workbench
2. BOM management supports creating new parts directly from the BOM workflow
3. Parts can be assigned one or more vendors with purchase-level controls
4. Lead times and safety stock are managed at the item-vendor purchasing level
5. Work orders are created from sales orders through a controlled conversion flow
6. Subassemblies are properly supported for shop floor and production planning
7. Sales order conversion supports both manufacturing work orders and service orders
8. Permissions and audit/change tracking are strengthened across all critical actions

Do not patch superficially. Make the architecture correct.

==================================================
CURRENT KNOWN GAPS TO FIX
==================================================

The existing codebase currently has these problems:
- MRP/planning and purchasing are split into separate modules and routes
- item purchasing fields are too generic and live only on the item master
- no item-vendor purchasing table exists
- BOM lines only select existing items and cannot create new parts inline
- no subassembly policy exists
- work orders link to sales orders, but there is no robust sales-order-to-work-order conversion service
- no service order entity exists
- audit logging exists but is not consistently enforced
- permissions exist only as a simple role JSON array and need stronger enforcement patterns

==================================================
ARCHITECTURE DECISIONS
==================================================

Keep:
- React frontend
- Express backend
- Drizzle ORM
- PostgreSQL
- current modular route structure as a base

Refactor toward:
- domain services for planning, purchasing, BOM, and production conversion
- shared transaction-safe service functions
- stronger audit logging wrapper for all create/update/convert/release actions
- permission guards by module + action
